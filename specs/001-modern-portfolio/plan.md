# Implementation Plan: Modern Portfolio Website

**Branch**: `001-modern-portfolio` | **Date**: 10 December 2025 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-modern-portfolio/spec.md`

## Summary

Build a modern, secure personal portfolio website that showcases Carlos's professional credentials as a Lead Platform Engineer. The site will feature:

1. **Professional CV & Credentials Section** - Immediate credibility with career history, experience, and expertise
2. **Project Portfolio** - Showcase past work from PlayStation, Novabase, and other professional engagements
3. **Security Foundation** - Zero hardcoded secrets, protection against common web vulnerabilities
4. **Performance & Accessibility** - Sub-4s load times, WCAG 2.1 AA compliance, responsive design
5. **Modern Design** - Contemporary visual presentation that reflects professional caliber

## Technical Context

**Language/Version**: JavaScript ES2020 (React 18.2, Node 16+)  
**Primary Dependencies**: React 18.2, React Router 6.14, Primer React 35.27, Bootstrap 5.3, Styled Components 6.0  
**Storage**: Static site - data in React components, JSON, or Markdown files (no database)  
**Testing**: React Testing Library, Jest (react-scripts test suite)  
**Target Platform**: Static web - GitHub Pages deployment or equivalent  
**Project Type**: Single frontend React application  
**Performance Goals**: 
- Homepage LCP < 2.5s (broadband), < 4s (4G)
- Core Web Vitals: All scores > 90
- Time to Interactive < 2s (broadband), < 4s (4G)

**Constraints**:
- Zero secrets in codebase or built assets
- HTTPS only (provided by hosting)
- Responsive: 320px mobile to 1920px+ desktop
- WCAG 2.1 AA accessibility minimum

**Scale/Scope**: 5-8 main pages, ~500 LOC per page average, ~50 KB gzipped bundle target

## Key Decisions

### 1. Data Management Strategy
**Decision**: JSON-based data files in `src/assets/` (cv-data.json, projects-data.json)

**Rationale**:
- No backend server required ✓
- Static site deployment to GitHub Pages ✓
- Easy to version control ✓
- Secure - no runtime secrets exposure ✓

**Alternative Rejected**: API-based approach adds complexity without benefit for static portfolio

### 2. Security Approach
**Decision**: Multi-layered security implementation:
- Input sanitization (prevent XSS)
- Security headers (CSP, X-Frame-Options)
- Environment variables for config (no secrets in code)
- Dependency scanning in CI/CD

**Rationale**:
- Meets requirement: "no secrets exist in codebase" ✓
- Protects against OWASP Top 10 ✓
- Static site inherently secure (no backend exposure) ✓

### 3. Responsive Design
**Decision**: Mobile-first approach with Bootstrap 5.3 + Primer React

**Rationale**:
- Existing dependency (Bootstrap already in package.json) ✓
- Primer React provides accessible components ✓
- Responsive helpers (react-responsive) already installed ✓

### 4. Performance Strategy
**Decision**: Code splitting by route, image optimization, lazy loading

**Rationale**:
- React Router already supports lazy loading ✓
- Existing build process (react-scripts) handles optimization ✓
- Target: < 4s load time on 4G ✓
